## Requisiti Hardware

Gentoo può essere installato su ogni sorta di sistemi, perciò elencare un minimo di caratteristiche hardware significa diminuire i tempi di attesa dell' utente. Qualsiasi macchina ragionevolmente recente può completare il processo di installazione in meno di un ora, ma non è una corsa quindi ci si prenda il proprio tempo, e si ricordi che ci mette quanto ci mette.

Per un processo di installazione tranquillo tuttavia, il manuale raccomanda almeno 40 GB di spazio per la partizione di root.

## Supporto di installazione di Gentoo Linux

**Suggerimento**

Anche se è consigliato usare un supporto di avvio ufficiale di Gentoo, è possibile utilizzare altri ambienti di installazione. Tuttavia, non si garantisce che contengano i componenti necessari. Se si usa un ambiente di installazione alternativo, si passi a [Preparare i dischi](/wiki/Handbook:HPPA/Installation/Disks/it "Handbook:HPPA/Installation/Disks/it").

### CD d'installazione minimale

Il _CD di installazione minimale_, anche noto come installcd, è una piccola immagine avviabile: un ambiente Gentoo autonomo. Questa immagine è curata dagli [sviluppatori di Gentoo](/wiki/Project:RelEng "Project:RelEng") e progettata per permettere a qualsiasi utente con una connessione ad Internet di installare Gentoo. Durante il processo di avvio, viene rilevato l'hardware, e vengono caricati automaticamente i driver appropriati.

Le versioni del CD di installazione minimale sono denominate secondo il formato:
install-<arch>-minimal-<release timestamp>.iso.

**Importante**

Il CD di installazione minimale necessita di almeno 140MB di RAM per avviarsi.

### Cosa sono i file stage?

Un [file stage](/wiki/Stage_file "Stage file") è un archivio che serve come germe per un ambiente Gentoo.

I file stage 3 si possono scaricare da releases/hppa/autobuilds/ su uno qualsiasi dei [mirror ufficiali di Gentoo](https://www.gentoo.org/downloads/mirrors/). Gli stage vengono aggiornati frequentemente e non sono perciò inclusi nelle immagini live ufficiali.

**Suggerimento**

Per ora, i _file stage_ possono essere ignorati. Saranno descritti con maggior dettaglio quando saranno necessari

**Nota**

Storicamente Gentoo ha fornito file stage minori di 3. Nel tempo questo tipo di installazioni sono diventate obsolete a causa di metodi migliori e per questa ragione non non sono stati più forniti.

## Scaricamento

### Scaricare il supporto

I mezzi di installazione predefiniti utilizzati da Gentoo Linux sono i _CD minimali di installazione_, che forniscono un ambiente Gentoo Linux avviabile e molto piccolo. Questo ambiente contiene tutti gli strumenti necessari per installare Gentoo. Le stesse immagini dei CD si possono scaricare dalla [pagina dei download](https://www.gentoo.org/downloads/) (raccomandata) o navigando autonomamente fino alla posizione del file ISO su uno dei tanti [mirror disponibili](https://www.gentoo.org/downloads/mirrors/).

#### Esplorare i mirror di Gentoo

Se si scarica da un mirror, il CD di installazione minimale si potrà trovare così:

1. Collegarsi al mirror, solitamente utilizzandone uno locale trovato sui [mirror dei sorgenti di Gentoo](https://www.gentoo.org/downloads/mirrors/).
2. Andare nella cartella releases/.
3. Scegliere la cartella relativa alla propria architettura (come **hppa/**).
4. Scegliere la cartella autobuilds/.
5. Per le architetture **amd64** e **x86** scegliere rispettivamente la cartella current-install-amd64-minimal/ o la cartella current-install-x86-minimal/. Per tutte le altre architetture entrare nella cartella current-iso/.

**Nota**

Alcune architetture come **arm**, **mips**, e **s390** non hanno un CD d'installazione minimale. Al momento il [Progetto di elaborazione delle versioni di Gentoo](/wiki/Project:RelEng "Project:RelEng") non supporta la creazione di file .iso per queste architetture.

Dentro questa posizione, il file immagine per l'installazione è quello con il suffisso .iso. Per esempio, si dia un'occhiata alla seguente lista:

```
[TXT]	install-amd64-minimal-20231217T170203Z.iso.asc	        2023-12-17 20:01        488
[TXT]	install-amd64-minimal-20231224T164659Z.iso.asc	        2023-12-24 20:41        488
[TXT]	install-amd64-minimal-20231231T163203Z.iso.asc	        2023-12-31 19:01        488
```

Nell'esempio di cui sopra, il file install-amd64-minimal-20240107T170309Z.iso è proprio il CD di installazione minimale. Ma come si può osservare, esistono anche altri file correlati:

- Un file .CONTENTS.gz che è un file di testo, compresso in formato gz, con una lista di tutti i file disponibili nel supporto di installazione. Può essere molto utile per verificare se alcuni driver o firmware sono disponibili in tale supporto prima di scaricarlo.
- Un file .DIGESTS che contiene i checksum del file ISO stesso, in vari formati/algoritmi. Può essere utilizzato per verificare l'integrità del file ISO.
- Un file .asc che è una firma crittografica del file ISO. Può essere usato per verificare l'integrità e l'autenticità dell'immagine - che il file scaricato sia davvero fornito dal [gruppo di elaborazione delle versioni di Gentoo](/wiki/Project:RelEng "Project:RelEng"), che non sia manomesso.

Si ignorino per ora gli altri file disponibili in questa posizione - si ripresenteranno con il procedere dell'installazione. Scaricare il file .iso e, se si desidera la verifica di quanto scaricato, scaricare anche il file .iso.asc relativo al file .iso.

**Suggerimento**

Il file .DIGESTS è necessario solo se la firma nel file .iso.asc non è verificata.

### Verificare i file scaricati

**Suggerimento**

Questo è un passaggio facoltativo e non è necessario per l'installazione di Gentoo Linux. Tuttavia, è consigliato poichè assicura che il file scaricato non sia corrotto e sia effettivamente fornito dalla [squadra dell'infrastruttura di Gentoo](/wiki/Project:Infrastructure "Project:Infrastructure").

Il file .asc fornisce una firma crittografica dell' ISO. Convalidandolo, ci si assicura che il file di installazione sia fornito proprio dalla squadra di elaborazione delle versioni di Gentoo e sia integro e immutato.

#### Verifica su un sistema Microsoft Windows

Per verificare prima la firma crittografica, è possibile utilizzare strumenti come [GPG4Win](https://www.gpg4win.org/). Dopo l'installazione, è necessario importare le chiavi pubbliche della squadra di elaborazione delle versioni di Gentoo. L'elenco delle chiavi è disponibile nella [pagina delle firme](https://www.gentoo.org/downloads/signatures/). Una volta importate, l'utente potrà così verificare la firma del file .asc.

#### Verifica su un sistema Linux

Su un sistema Linux, il metodo più comune per verificare la firma crittografica consiste nell'usare il programma [app-crypt/gnupg](https://packages.gentoo.org/packages/app-crypt/gnupg). Con questo pacchetto installato, può essere usato il seguente comando per verificare la firma del file .asc.

**Suggerimento**

Importando le chiavi di Gentoo, controllare che l'identificativo (ID) della chiave di 16 caratteri ( `BB572E0E2D182910`) corrisponda.

Le chiavi di Gentoo possono essere scaricate da hkps://keys.gentoo.org usando le impronte disponibili sulla [pagina delle firme](https://www.gentoo.org/downloads/signatures/):

`user $` `gpg --keyserver hkps://keys.gentoo.org --recv-keys 13EBBDBEDE7A12775DFDB1BABB572E0E2D182910`


## Scrivere il supporto di avvio

Naturalmente, solo scaricando il file ISO, l'installazione di Gentoo Linux non può iniziare. Il file ISO deve essere scritto su un supporto avviabile. Questo normalmente richiede che l'immagine sia estratta in un filesystem, o scritta direttamente su un dispositivo.

### Scrittura su un dispositivo USB avviabile

**Attenzione**

Usare strumenti come Ventoy per scrivere su una chiavetta USB può provocare problemi di avvio.

La maggior parte dei sistemi moderni supporta l' avvio da un dispositivo USB.

#### Scrivere con Linux

dd è solitamente disponibile nella maggior parte delle distribuzioni Linux, e può essere usato per scrivere il supporto di avvio di Gentoo su una chiavetta USB.

##### Stabilire il percorso del dispositivo USB

Prima di scrivere, deve essere stabilito il percorso del dipositivo di memorizzazione desiderato.

dmesg mostrerà informazioni dettagliate descrivendo il dispositivo di memorizzazione al suo inserimento nel sistema:

`root #` `dmesg`

```
[268385.319745] sd 19:0:0:0: [sdd] 60628992 512-byte logical blocks: (31.0 GB/28.9 GiB)
```

In alternativa, si può usare lsblk per mostrare i dispositivi di memorizzazione disponibili:

`root #` `lsblk`

```
sdd           8:48   1  28.9G  0 disk
├─sdd1        8:49   1   246K  0 part
├─sdd2        8:50   1   2.8M  0 part
├─sdd3        8:51   1 463.5M  0 part
└─sdd4        8:52   1   300K  0 part

```

Una volta stabilito il nome del dispositivo, lo si può aggiungere al prefisso _/dev/_ del percorso per ottenere il percorso del dispositivo /dev/sdd.

**Suggerimento**

Usare il percorso base del dispositivo, p.es.sdd invece di sdd1 è consigliato dato che il _supporto di avvio_ di Gentoo contiene uno schema delle partizioni [GPT](/wiki/GPT "GPT") completo.

##### Scrivere con dd

**Attenzione**

Ci si assicuri di controllare il percorso di destinazione ( _of=target_) prima di eseguire dd, perché verrà sovrascritto.

Col percorso del dispositivo (/dev/sdd) e il _supporto di avvio_ install-amd64-minimal-<release timestamp>.iso pronti:

`root #` `dd if=install-amd64-minimal-<release timestamp>.iso of=/dev/sdd bs=4096 status=progress && sync`

**Nota**

**if=** specifica il _file di input_, **of=** specifica il _file di output_, che in questo caso, è il dispositivo.

**Suggerimento**

**bs=4096** è usato poichè nella maggior parte dei casi velocizza i trasferimenti, **status=progress** mostra le statistiche dei trasferimenti.

#### Scrittura con Windows

[win32diskimager](https://sourceforge.net/projects/win32diskimager/) è un semplice programma di utilità per Windows per scrivere immagini iso su una chiavetta USB.

E' corredato anche da una eccellente documentazione di supporto se necessaria.

**Attenzione**

Scaricare solo da [https://sourceforge.net/projects/win32diskimager/](https://sourceforge.net/projects/win32diskimager/)

### Masterizzare un disco

**See also**

Delle istruzioni più dettagliate possono essere trovate in [CD/DVD/BD\_writing#Image\_writing](/wiki/CD/DVD/BD_writing#Image_writing.2Fit "CD/DVD/BD writing").

#### Masterizzare con Microsoft Windows 7 e successivi

Le versioni di Microsoft Windows 7 e successive possono montare e masterizzare immagini ISO su supporti ottici senza la necessità di software di terze parti. Basta inserire un disco masterizzabile, cercare i file ISO scaricati, fare clic con il pulsante destro del mouse sul file in Esplora risorse e selezionare "Masterizza immagine disco".

#### Masterizzare con Linux

Il programma di utilità cdrecord dal pacchetto [app-cdr/cdrtools](https://packages.gentoo.org/packages/app-cdr/cdrtools) può masterizzare immagini ISO su Linux.

Per masterizzare il file ISO sul CD nel percorso /dev/sr0 (ciò corrisponde al primo masterizzatore CD nel sistema - sostituirlo con il dispositivo corretto se necessario):

`user $` `cdrecord dev=/dev/sr0 install-hppa-minimal-20141204.iso`

Gli utenti che preferiscono un'interfaccia grafica possono usare K3B, incluso nel pacchetto [kde-apps/k3b](https://packages.gentoo.org/packages/kde-apps/k3b). In K3B, andare su Strumenti e scegliere Masterizza immagine CD.

### Accesso di root della LiveGUI

sudo è stato configurato per funzionare senza il bisogno di una password sulla LiveGUI poiché sia root che gentoo hanno una password cifrata.

Per ottenere l'accesso all'utente superuser, in qualsiasi terminale eseguire:

`root #` `sudo -i`

### Configurazione hardware aggiuntiva

Quando il supporto di installazione si avvia, rileva tutti i dispositivi hardware e carica i moduli del kernel appropriati per supportare quell'hardware. Nella maggior parte dei casi fa un ottimo lavoro. Tuttavia, in alcuni casi, può non caricare automaticamente i moduli del kernel necessari al sistema. Se il rilevamento automatico delle periferiche PCI non riconosce qualche componente hardware, i moduli del kernel appropriati dovranno essere caricati manualmente.

Nel prossimo esempio viene caricato il modulo 8139too (che supporta certi tipi di interfacce di rete):

`root #` `modprobe 8139too`

### Facoltativo: Account utenti

Se altre persone hanno bisogno di accedere all'ambiente di installazione o c'è la necessità di eseguire comandi come utenti non-root dal supporto di installazione (ad esempio per chattare utilizzando irssi senza privilegi di root per ragioni di sicurezza), allora è necessario creare un altro account utente ed impostare una password di root complessa.

Per cambiare la password di root, usare il comando passwd:

`root #` `passwd`

```
New password: (Inserisci la nuova password)
Re-enter password: (Riscrivi la nuova password)

```

Per creare un account utente, prima inserire le sue credenziali e poi la sua password. I comandi useradd e passwd servono per queste due operazioni.

Nel prossimo esempio, viene creato un utente chiamato "luca" e aggiunto ai gruppi `users` e `wheel`:

`root #` `useradd -m -G users,wheel luca
`

`root #` `passwd luca`

```
New password: (Inserisci la password di Luca)
Re-enter password: (Riscrivi la password di Luca)

```

Per passare dall' utente _root_(attuale) a quello appena creato, usare il comando su:

`root #` `su - luca`

### Facoltativo: Consultare la documentazione durante l'installazione

#### Terminali

Per consultare la guida da un terminale durante l'installazione, prima creare un account utente come descritto sopra, poi premere `Alt` + `F2` per passare ad un nuovo terminale ed effettuare l'accesso col nuovo utente creato. Seguendo il [principio del privilegio minimo](https://en.wikipedia.org/wiki/Principle_of_least_privilege "wikipedia:Principle of least privilege"), sarebbe ottima abitudine evitare di esplorare il web o genericamente eseguire qualunque operazione con privilegi maggiori del necessario. L'account root ha il controllo completo del sistema e perciò va usato con moderazione.

Durante l'installazione, il browser web links può essere usato per sfogliare il manuale Gentoo - naturalmente solo dal momento in cui la connessione a Internet è funzionante.

`user $` `links https://wiki.gentoo.org/wiki/Handbook:HPPA/it`

Per tornare al terminale di origine, premere `Alt` + `F1`.

**Suggerimento**

Quando l'ambiente minimale di Gentoo o l'ambiente amministrativo di Gentoo sono avviati, sette terminali sono disponibili. Si può passare da uno all'altro premendo `Alt` e poi un tasto funzione tra `F1`- `F7`. Potrebbe essere utile spostarsi in un nuovo terminale mentre si aspetta la fine di un lavoro, per aprire la documentazione, etc.

#### GNU Screen

Il programma di utilità [Screen](/wiki/Screen "Screen") è preinstallato nel supporto di installazione ufficiale di Gentoo. Potrebbe risultare più efficiente, per gli appassionati di lunga data di Linux, usare screen per visualizzare le istruzioni di installazione in più pannelli anziché con il metodo dei terminali multipli appena esposto.

### Facoltativo: Avviare il servizio SSH

Per consentire ad altri utenti di accedere al sistema durante l'installazione (magari per dare/ricevere assistenza durante un installazione, o per farla a distanza), si deve creare un account utente (come descritto sopra) ed avviare il servizio SSH.

Per avviare il servizio SSH su un sistema che usa OpenRC come sistema di init, eseguire il comando seguente:

`root #` `rc-service sshd start`

**Nota**

Se gli utenti accedono al sistema, vedranno un messaggio che indica che è necessario confermare la chiave del computer host (attraverso la cosiddetta impronta digitale). Ciò è normale dato che si accede al sistema per la prima volta. Tuttavia, in seguito, quando il sistema sarà configurato e qualcuno accederà al sistema appena installato, il client SSH avviserà che il codice del computer host è cambiato. Ciò avviene perché l'utente farà l'accesso - per SSH - su un nuovo server (cioè sul nuovo sistema appena installato piuttosto che sull'ambiente di installazione che si sta utilizzando). Seguire le istruzioni date a schermo per sostituire la chiave host sul sistema client.

Per poter usare sshd, la rete deve funzionare correttamente. Proseguire con il capitolo su come [configurare la rete](/wiki/Handbook:HPPA/Installation/Networking/it "Handbook:HPPA/Installation/Networking/it").
